01

Driver Overview

Device Node/dev/sanath_timer
Buffer16 rows × 128 bytes (ROW_SIZE × MEM_SIZE)
Timer Interval1000 ms
Synchronizationspinlock_t — spin_lock_irqsave / spin_unlock_irqrestore
IOCTL Magic'f'
IOCTL CommandsSTART_TIMER (_IO 'f',1) · STOP_TIMER (_IO 'f',2)
File Operationsopen · release · read · unlocked_ioctl

A periodic kernel timer fires every 1000 ms and writes timer_event_N\n into the ring buffer. Userspace reads events via the device node. The timer is started and stopped via ioctl from a userspace control program.

02

Spinlock vs Mutex — Context Matters

The timer callback runs in softirq context. Mutexes can sleep — they are illegal in softirq context. Spinlocks are non-sleeping and safe.

PRIMITIVECAN SLEEPSAFE IN SOFTIRQ
struct mutexYES — illegalNO
spinlock_tNOYES

// spin_lock_irqsave explained

flags stores the CPU interrupt state before the lock is acquired. spin_unlock_irqrestore restores it exactly — not blindly re-enabling. This is a macro, not a function, which is why flags is passed by value rather than by pointer.

unsigned long flags;
spin_lock_irqsave(&kernel_logger.lock, flags);   /* saves irq state + acquires */
    /* critical section — shared data only */
spin_unlock_irqrestore(&kernel_logger.lock, flags); /* releases + restores irq state */
// GOLDEN RULE

Keep the spinlock critical section as small as possible. Only shared data structure manipulation belongs inside. Everything else — wake_up, mod_timer, copy_to_user — goes outside after releasing.

03

What Cannot Be Called Under Spinlock

FUNCTIONREASONSAFE UNDER SPINLOCK
wake_up_interruptibleTouches scheduler — has own locksNO — call after unlock
copy_to_userCan page fault — sleepsNEVER
kmalloc(GFP_KERNEL)Can sleep waiting for memoryNEVER
mod_timerHas own timer subsystem locksRisky — call after unlock
memcpy / snprintfNo locks, no sleepYES

// Correct pattern in timer_callback

spin_lock_irqsave(&kernel_logger.lock, flags);
    snprintf(buffer[write_idx], MEM_SIZE, "timer_event_%ld\n", count++);
    write_idx = (write_idx + 1) % ROW_SIZE;
    /* update count / read_indexer */
    is_active = kernel_logger.timer_active;
spin_unlock_irqrestore(&kernel_logger.lock, flags);

wake_up_interruptible(&etx_wait_queue);     /* AFTER unlock */
if (is_active)
    mod_timer(&etx_timer, jiffies + msecs_to_jiffies(TIMEOUT)); /* AFTER unlock */
04

copy_to_user Must Not Be Called Under Spinlock

copy_to_user can trigger a page fault if the userspace page is swapped out. Page faults require sleeping — illegal under spinlock. This was causing Bad address (EFAULT) errors on cat /dev/sanath_timer.

// Fix — copy to local tmp buffer inside spinlock, copy_to_user outside

uint8_t tmp[MEM_SIZE];

spin_lock_irqsave(&kernel_logger.lock, flags);
    memcpy(tmp, kernel_logger.kernel_buffer[read_idx], copy_len);  /* safe */
    memset(kernel_logger.kernel_buffer[read_idx], 0, MEM_SIZE);
    read_idx = (read_idx + 1) % ROW_SIZE;
    count--;
spin_unlock_irqrestore(&kernel_logger.lock, flags);

if (copy_to_user(buf, tmp, copy_len))   /* safe — outside spinlock */
    return -EFAULT;
05

Timer Active Flag — Stopping a Periodic Timer

A one-shot kernel timer becomes periodic by calling mod_timer at the end of the callback. del_timer_sync stops the pending timer but if the callback is already executing, it will complete — and call mod_timer again, re-arming the timer after stop. This caused events to keep appearing after STOP_TIMER.

// BUG — timer kept firing after STOP_TIMER

del_timer_sync was called but the callback unconditionally called mod_timer at the end, re-arming the timer. Only timer_event_3 was visible after cat because the buffer wrapped and older events were overwritten.

// Fix — timer_active flag in struct

/* in struct */
int timer_active;

/* START_TIMER ioctl */
kernel_logger.timer_active = 1;
mod_timer(&etx_timer, jiffies + msecs_to_jiffies(TIMEOUT));

/* STOP_TIMER ioctl */
kernel_logger.timer_active = 0;
del_timer_sync(&etx_timer);

/* in callback — only re-arm if still active */
if (is_active)
    mod_timer(&etx_timer, jiffies + msecs_to_jiffies(TIMEOUT));
06

del_timer vs del_timer_sync

FUNCTIONBEHAVIOURUSE WHEN
del_timerCancels pending timer. Does NOT wait for running callback.Never in driver exit
del_timer_syncCancels AND waits for any running callback to complete.Always — exit and STOP_TIMER

Using del_timer in exit means the module unloads while a callback is still executing — use-after-free / kernel panic.

07

Return Actual String Length, Not MEM_SIZE

Returning MEM_SIZE (128) bytes when the actual message is 14 bytes (timer_event_0\n) caused cat to receive 128 bytes of data with 114 null bytes appended, resulting in display corruption and segfaults in userspace readers.

// Fix — use strnlen inside the spinlock after count > 0 is confirmed

/* compute AFTER the while loop — inside spinlock — count > 0 guaranteed */
msg_len  = strnlen(kernel_logger.kernel_buffer[read_idx], MEM_SIZE);
copy_len = min(len, msg_len);
memcpy(tmp, kernel_logger.kernel_buffer[read_idx], copy_len);
...
return copy_len;   /* return actual bytes, not MEM_SIZE */
// CRITICAL — compute msg_len AFTER waiting, not before

Computing msg_len before the wait loop reads an empty/zeroed slot — strnlen returns 0 — copy_len stays 0 — read returns 0 even after waking with valid data. Always compute lengths inside the spinlock after count > 0 is confirmed.

08

All Bugs Encountered & Fixed

BUGFIX
write_indexer never incremented in timer callback — all events written to slot 0 write_indexer = (write_indexer + 1) % ROW_SIZE after snprintf
timer kept firing after STOP_TIMER — buffer wrapping, only last event visible timer_active flag — guard mod_timer in callback with if (is_active)
del_timer in exit — use after free if callback running during rmmod del_timer_sync in exit and STOP_TIMER ioctl
copy_to_user called under spinlock — EFAULT / Bad address memcpy to tmp under spinlock, copy_to_user after unlock
wake_up_interruptible called under spinlock — scheduler lock conflict capture is_active under spinlock, call wake_up and mod_timer after unlock
msg_len computed before wait — returns 0 on first read when buffer was empty compute strnlen inside spinlock after while loop confirms count > 0
read always returned MEM_SIZE (128) — null bytes corrupted cat output return copy_len (actual string length) not MEM_SIZE
